**2. Create `apps/web/components/platform/drive/drive-file-table.tsx`:**
A `"use client"` component that builds directly on `@tanstack/react-table` (NOT the shared DataTable component, which lacks row selection). Accept props: `files: DriveFile[]`, `onFileClick: (file: DriveFile) => void`, `onSelectionChange: (selectedIds: string[]) => void`, `onDeleteFile: (file: DriveFile) => void`.

Build the table with `useReactTable` from `@tanstack/react-table` using `getCoreRowModel`, `getPaginationRowModel`, and `getFilteredRowModel`. Enable row selection with `enableRowSelection: true` and the `onRowSelectionChange` handler.

Define columns in this exact order:
1. **Select** -- Checkbox column. Header: Checkbox that toggles all rows. Cell: Checkbox for individual row. Use `Checkbox` from `@workspace/ui/components/checkbox`. Stop propagation on checkbox click so it does not trigger row click.
2. **Nome** -- Display `file.name` truncated with `truncate max-w-[200px]` and wrap in a `Tooltip` showing full file name. Header text: `Nome`.
3. **Tipo** -- Display file type icon using a mapping function. Use these lucide-react icons: `FileTextIcon` for `"pdf"`, `ImageIcon` for `"image"`, `FileTypeIcon` for `"word"`, `FileSpreadsheetIcon` for `"excel"`, `FileIcon` for `"other"`. Next to the icon, show `FILE_TYPE_LABELS[file.type]` from drive-data. Header text: `Tipo`.
4. **Tamanho** -- Display `formatFileSize(file.size)` from drive-data. Use `font-mono text-sm` class. Header text: `Tamanho`.
5. **Data de upload** -- Display `new Date(file.uploadedAt).toLocaleDateString("pt-BR")`. Header text: `Data`.
6. **Autor** -- Display `Avatar` with `AvatarFallback` showing `file.author.initials`, plus author name text. Header text: `Autor`.
7. **Acoes** -- Render `DriveFileRowActions` component. No header text.

Use `Table`, `TableBody`, `TableCell`, `TableHead`, `TableHeader`, `TableRow` from `@workspace/ui/components/table` for rendering.

Make the entire row clickable (calls `onFileClick(file)`) EXCEPT when clicking the checkbox or actions column. Apply `cursor-pointer hover:bg-muted/50` to rows. Hide columns `Tamanho` and `Autor` on mobile using the class `hidden md:table-cell`.

When there are selected rows, call `onSelectionChange` with the array of selected file IDs whenever selection changes.

Handle the delete flow: when `DriveFileRowActions` triggers `onDelete`, show an `AlertDialog` with title `Excluir arquivo` and description `Tem certeza que deseja excluir este arquivo? Esta acao nao pode ser desfeita.` with `Cancelar` and `Excluir` buttons. On confirm, call `onDeleteFile`.

Also handle bulk delete: accept optional `onBulkDelete: (fileIds: string[]) => void`. The `DriveToolbar` in the parent will call this via a callback.

If `files` is empty, render `DriveEmptyState variant="folder"`.

**2. Create `apps/web/components/platform/drive/drive-upload-handler.tsx`:**
A `"use client"` component that exports two functions as a custom hook `useUploadHandler()`:
- `triggerUpload()` -- Creates a hidden `input[type=file]` element, sets `multiple` attribute, clicks it. On file selection, calls `simulateUpload(files)`.
- `simulateUpload(files: FileList)` -- Uses `toast.loading` from sonner with a fixed toast ID `"drive-upload"`. For single file: show `Enviando arquivo...`, after 1500ms show `toast.success("Upload concluido", { id: "drive-upload" })`. For multiple files: show `Enviando 1 de {total} arquivos...`, update every 800ms incrementing the counter, then show `toast.success("Upload concluido", { id: "drive-upload" })`.

Also export `triggerDownload(file: DriveFile)` -- Uses `toast.loading("Preparando download...", { id: "drive-download" })`, after 1500ms calls `toast.success("Download concluido", { id: "drive-download" })`.

Also export `triggerBulkDownload(count: number)` -- Uses `toast.loading("Preparando download (ZIP)...", { id: "drive-bulk-download" })`, after 2000ms calls `toast.success("Download concluido", { id: "drive-bulk-download" })`.
