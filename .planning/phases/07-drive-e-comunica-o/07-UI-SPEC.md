## Visual Hierarchy

| Screen | Primary focal point | Secondary elements | Notes |
|--------|---------------------|--------------------|-------|
| Drive main | Tabs (Oportunidades / Obras) at the top with breadcrumb navigation below, folder list as the main content | Toolbar with search + action buttons, folder metadata (date, file count), 3-dot action menus | Tabs define the primary segmentation; the folder list must feel like a directory browser, not a generic card grid. |
| Drive folder contents | Toolbar (search + Nova pasta + Upload) above the file DataTable | Subfolder cards at the top, file rows with type icons and author avatars, multi-select checkbox column | When inside a folder, the breadcrumb updates to show full path. Subfolders appear above the file table as a distinct row. |
| Drive file preview | Sheet slide-over panel from the right showing file metadata | File type icon, name, size, date, author info, action buttons (Abrir, Download) | The panel overlays the listing without replacing it. Mobile uses a bottom sheet instead. |
| Comunicacao mural | Card feed in vertical list with highlighted (destaque) cards pinned at the top | Toolbar with category dropdown, period selector, search, and "Novo comunicado" CTA | Destaques must be visually distinct (border/background emphasis) and always sort above regular cards. |
| Comunicado detail | Full content area with title, body text, and metadata sidebar (category badge, date, author) | Edit and Excluir action buttons, voltar navigation link | This is a dedicated page at `/comunicacao/[id]`, not a modal. |
| Comunicado publish wizard | 2-step Dialog modal with step indicator in header | Step 1: titulo + categoria fields. Step 2: conteudo textarea + destaque checkbox | The wizard is a Dialog, not a full page. Footer shows navigation (Voltar/Proximo/Publicar). |

Icon-only actions are not allowed in this phase. Every visible action must ship with a text label.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.15 |

Use sentence case in headings, field labels, badges, and actions. Reserve display text for page titles ("Drive" and "Comunicacao"). File names in the DataTable use Body weight 400 at 14px (Label size, regular weight) to keep the table compact. Folder names use Label weight 600 at 14px.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` / `oklch(1 0 0)` | Page background, file table canvas, feed card backgrounds, Sheet panel background |
| Secondary (30%) | `var(--secondary)` / `oklch(0.967 0.001 286.375)` | Toolbar area, tab list background, folder rows hover, mural card borders, Drive breadcrumb area |
| Accent (10%) | `var(--primary)` / `oklch(0.505 0.213 27.518)` | Primary CTAs (Upload, Nova pasta, Novo comunicado, Publicar), active tab indicator, destaque card border/highlight, selected file checkbox state |
| Destructive | `var(--destructive)` / `oklch(0.577 0.245 27.325)` | Delete confirmation actions, file upload error state, exclusion buttons only |

Accent reserved for: the `Fazer upload` CTA, `Nova pasta` CTA, `Novo comunicado` CTA, `Publicar` action in the wizard, the active tab underline in Drive, the destaque visual emphasis on pinned comunicado cards, and the selected state of file checkboxes. Do not use accent for every icon, file type badge, or neutral metadata element.

60/30/10 split: 60% dominant surfaces, 30% structural secondary surfaces, 10% accent emphasis.

Category badge colors for Comunicacao use distinct semantic tones within the existing palette:
- Geral: `secondary` variant (muted gray)
- Operacional: `default` variant (primary brand tone)
- Comercial: `outline` variant (outlined neutral)
- RH: custom warm tone via inline `className` with `bg-amber-100 text-amber-800 dark:bg-amber-900 dark:text-amber-200`
- TI: custom cool tone via inline `className` with `bg-blue-100 text-blue-800 dark:bg-blue-900 dark:text-blue-200`

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (Drive) | `Fazer upload` |
| Primary CTA (Comunicacao) | `Novo comunicado` |
| Drive empty section | `Nenhum arquivo encontrado` / `Faca upload do primeiro arquivo para iniciar.` |
| Drive empty folder | `Esta pasta esta vazia` / `Faca upload de um arquivo para comecar.` |
| Drive no search results | `Nenhum resultado para sua busca` / `Tente ajustar os termos de busca.` |
| Comunicacao empty state heading | `Nenhum comunicado publicado` |
| Comunicacao empty state body | `Publique o primeiro comunicado para informar a equipe.` |
| Comunicacao no search results | `Nenhum comunicado encontrado` / `Ajuste os filtros ou termos de busca.` |
| Error state | `Nao foi possivel carregar este conteudo agora. Atualize a pagina ou volte para a listagem anterior.` |
| Delete file confirmation | `Excluir arquivo`: `Tem certeza que deseja excluir este arquivo? Esta acao nao pode ser desfeita.` |
| Delete folder confirmation | `Excluir pasta`: `Tem certeza que deseja excluir esta pasta e todos os seus arquivos?` |
| Delete comunicado confirmation | `Excluir comunicado`: `Tem certeza que deseja excluir este comunicado? Esta acao nao pode ser desfeita.` |
| Bulk delete confirmation | `Excluir arquivos`: `Tem certeza que deseja excluir os {count} arquivos selecionados?` |
| Upload single toast | `Enviando arquivo...` then `Upload concluido` |
| Upload multiple toast | `Enviando {current} de {total} arquivos...` then `Upload concluido` |
| Upload error toast | `Falha no envio` (inline retry: `Tentar novamente`) |
| Download toast | `Preparando download...` then `Download concluido` |
| Bulk download toast | `Preparando download (ZIP)...` then `Download concluido` |
| Publish wizard step 1 title | `Novo comunicado - Informacoes` |
| Publish wizard step 2 title | `Novo comunicado - Conteudo` |
| Publish success toast | `Comunicado publicado com sucesso` |
| Draft save toast | `Rascunho salvo` |

Supporting copy rules:
- Drive tab labels are `Oportunidades` and `Obras`.
- Drive toolbar buttons are `Nova pasta` and `Upload`.
- Folder action menu items are `Upload`, `Renomear`, `Excluir`.
- File action menu items are `Download`, `Visualizar`, `Renomear`, `Excluir`.
- Multi-select toolbar shows `{count} selecionado(s)` with inline `Download` and `Excluir` actions.
- Comunicacao toolbar filter labels are `Todas as categorias` (default) and `Qualquer periodo` (default).
- Comunicacao wizard buttons: step 1 shows `Proximo`, step 2 shows `Voltar` and `Publicar`.
- Comunicacao wizard optional action: `Visualizar` opens preview modal, `Salvar rascunho` saves to localStorage.
- Comunicado detail actions: `Editar` and `Excluir`.
- Date format throughout: `DD/MM/AAAA` using `toLocaleDateString("pt-BR")`.
- File size format: bytes to KB/MB with one decimal (e.g., `2.4 MB`, `156.0 KB`).

## Interaction Contract

| Area | Contract |
|------|----------|
| Drive tabs | `/drive` opens with the first visible tab based on profile. Admin sees both tabs (default: Oportunidades). Operations sees only Obras. Commercial cannot access Drive (RestrictedModuleState). Tab switches preserve search query state. |
| Drive folder navigation | Clicking a folder enters it in client-side state (no URL change). The breadcrumb updates to show `Drive > [Secao] > [Pasta]`. Clicking a subfolder adds another level. Breadcrumb segments are clickable for backward navigation. |
| Drive folder list | Alphabetically sorted by name. Each row shows: folder name, creation date (DD/MM/AAAA), file count. Hover on folder row shows a Tooltip with entity info (opportunity/obra status and responsible). 3-dot menu per row: Upload, Renomear, Excluir. |
| Drive file DataTable | Columns: Checkbox (select), Nome (truncated + tooltip), Tipo (file type icon), Tamanho, Data de upload, Autor (avatar + name), Acoes (3-dot menu). Sorted by upload date descending. Row click opens the preview Sheet. |
| Drive multi-select | Checkbox column enables row selection. When one or more files are selected, the toolbar switches to show: `{count} selecionado(s)` counter + `Download` button + `Excluir` button. Deselecting all reverts the toolbar to the default state. |
| Drive file preview | Sheet from the right (desktop) or bottom (mobile). Shows: file type icon placeholder, file name, type label, size, upload date, author name. Action buttons: `Abrir` (disabled with "Preview nao disponivel" note) and `Download`. Close via X button or click outside. |
| Drive upload | `Upload` button opens native file picker. On selection, a toast appears with loading state. Single file: `Enviando arquivo...`. Multiple: `Enviando {n} de {total} arquivos...`. Completes with success toast. Error shows inline error status on the file row with `Tentar novamente` button. |
| Drive download | Single file: toast with `Preparando download...` then `Download concluido`. Bulk: toast with `Preparando download (ZIP)...` then `Download concluido`. All simulated, no real file transfer. |
| Drive mobile | Tabs remain at top. Folder list uses stacked cards instead of table rows. File list switches to stacked cards. Preview uses bottom Sheet. |
| Comunicacao mural | `/comunicacao` shows a card feed. Cards with `destaque: true` appear first with visual emphasis (accent border-left or subtle accent background). Regular cards follow, sorted by date descending. |
| Comunicacao card content | Each card shows: title (heading weight), resumo (2-line clamp), category badge (colored), date, author (avatar + name), and optional destaque badge. Entire card is clickable to navigate to `/comunicacao/[id]`. |
| Comunicacao toolbar | Desktop: category dropdown + period selector + search input + `Novo comunicado` button in one row. Mobile: stacks vertically, CTA stays prominent at top. |
| Comunicacao detail | `/comunicacao/[id]` shows full content with metadata (category badge, date, author avatar + name). Actions: `Editar` button navigates to `/comunicacao/[id]/editar`. `Excluir` button opens AlertDialog confirmation. `Voltar` link returns to mural. |
| Comunicacao edit | `/comunicacao/[id]/editar` reuses the same form layout as the publish wizard but in a full-page format. Pre-populated with existing data. Save action shows success toast and redirects to detail. |
| Comunicacao publish wizard | `Novo comunicado` button opens a Dialog. Step 1: titulo (Input, required) + categoria (Select, required). Step 2: conteudo (Textarea, required) + destaque (Checkbox). Footer: step 1 has `Proximo`, step 2 has `Voltar`, `Salvar rascunho`, `Visualizar`, and `Publicar`. Validation prevents advancing without required fields. |
| Comunicacao draft | `Salvar rascunho` saves current wizard state to localStorage under key `hi-comunicado-draft`. On reopening the wizard, if a draft exists, pre-populate fields and show a subtle indicator. Clear draft on successful publish. |
| Comunicacao publish success | Toast `Comunicado publicado com sucesso` + navigate to the new comunicado detail page. |

No drawer-heavy workflow is allowed in this phase. The comunicado detail and edit are full pages. The publish wizard uses a Dialog. The file preview uses a Sheet.
