**1. Create `apps/web/components/platform/comunicacao/comunicacao-publish-dialog.tsx`:**
A `"use client"` component that accepts `open: boolean` and `onOpenChange: (open: boolean) => void`. Render a `Dialog` from `@workspace/ui/components/dialog`.

**Internal state:**
- `step: 1 | 2` -- starts at 1
- `title: string` -- empty string default
- `category: ComunicadoCategory | ""` -- empty string default
- `content: string` -- empty string default
- `isDestaque: boolean` -- false default
- `showPreview: boolean` -- false default
- `errors: Record<string, string>` -- validation error messages

**Draft persistence:**
- On mount, check `localStorage.getItem("hi-comunicado-draft")`. If exists, parse JSON and pre-populate `title`, `category`, `content`, `isDestaque`. Show a subtle info text `Rascunho recuperado` in muted-foreground below the dialog title.
- `handleSaveDraft()` -- saves current state to localStorage as `JSON.stringify({ title, category, content, isDestaque })` under key `"hi-comunicado-draft"`. Shows `toast("Rascunho salvo")`.
- Clear draft on successful publish: `localStorage.removeItem("hi-comunicado-draft")`.

**Step 1 content:**
- `DialogHeader` with `DialogTitle` text `Novo comunicado - Informacoes`.
- `Input` for title with label `Titulo` and id `comunicado-title`. Required.
- `Select` for category with label `Categoria` and the options from `getComunicadoCategoryOptions()` (excluding the "all" option -- only show the 5 real categories). Placeholder `Selecione uma categoria`. Required.
- Validation: `title` must be non-empty, `category` must be selected. If invalid, set `errors` with Portuguese messages: `Titulo e obrigatorio`, `Categoria e obrigatoria`.

**Step 2 content:**
- `DialogHeader` with `DialogTitle` text `Novo comunicado - Conteudo`.
- `Textarea` for content with label `Conteudo`, id `comunicado-content`, and `rows={8}`. Required.
- `Checkbox` with label `Marcar como destaque` for `isDestaque`. Use the Checkbox component from `@workspace/ui/components/checkbox`.
- Validation: `content` must be non-empty. Error message: `Conteudo e obrigatorio`.

**Footer buttons:**
- Step 1: `Proximo` button (primary). Validates step 1 fields before advancing.
- Step 2: `Voltar` button (outline, calls `setStep(1)`), `Salvar rascunho` button (outline, calls `handleSaveDraft()`), `Visualizar` button (outline, sets `showPreview` to true), `Publicar` button (primary, validates and publishes).

**Preview:**
When `showPreview` is true, render a nested `Dialog` (or swap content) showing a preview card using `ComunicacaoCard`-like layout with the current form values. Include a `Fechar` button to dismiss.

**Publish flow:**
- Validate all fields (title, category, content).
- If valid: clear draft from localStorage, close dialog, show `toast.success("Comunicado publicado com sucesso")`, and navigate to `/comunicacao/${newId}` where `newId` is the first existing comunicado ID (simulated -- in mock, use `COMUNICADOS[0].id` as a standin since there is no real persistence).
- Use `useRouter().push()` for navigation.

**Reset:**
When dialog closes (onOpenChange to false), reset step to 1, clear errors. Do NOT clear form fields (they should persist for draft recovery).
