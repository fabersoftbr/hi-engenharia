    Actually, checking the UAT diagnosis: the main issue is that `z.number().min(1)` shows "Expected number" in English when the input is empty/NaN because the type error is not customized. The fix is to use `invalid_type_error` option.

    Implementation pattern for each field:
    ```typescript
    clientName: z.string({ invalid_type_error: "Nome e obrigatorio" }).min(1, "Nome e obrigatorio"),
    phone: z.string({ invalid_type_error: "Telefone e obrigatorio" }).min(1, "Telefone e obrigatorio"),
    city: z.string({ invalid_type_error: "Cidade e obrigatoria" }).min(1, "Cidade e obrigatoria"),
    monthlyConsumption: z.number({ invalid_type_error: "Consumo mensal e obrigatorio" }).min(1, "Consumo mensal e obrigatorio"),
    ```

    Note: The UAT mentioned missing accents (obrigatorio -> obrigatorio). However, "obrigatorio" without accent is acceptable in Portuguese. The user complaint was specifically about English messages appearing. Focus on adding `invalid_type_error` to suppress the "Expected number" message.
