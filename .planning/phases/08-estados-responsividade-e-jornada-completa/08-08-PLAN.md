Update the Card component in journey-card.tsx to use basis-0 alongside flex-1, ensuring cards have equal width regardless of content length.

Current code (line 34):
```tsx
<Card className="w-full min-w-[200px] flex-1">
```

Change to:
```tsx
<Card className="w-full min-w-[200px] flex-1 basis-0">
```

The basis-0 class forces flex items to start from zero width before flex-1 distributes available space equally. Without it, content length (like 2-digit vs 1-digit counts) drives the initial width, causing visual inconsistency.

This fixes the UAT-reported issue where cards with "10 itens ativos" were wider than cards with "5 itens ativos".
