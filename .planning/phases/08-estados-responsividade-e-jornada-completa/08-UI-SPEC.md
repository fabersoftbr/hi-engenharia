## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, gap between badge items |
| sm | 8px | Compact element spacing, toast internal padding |
| md | 16px | Default element spacing, card padding, main content `gap-4` |
| lg | 24px | Section padding, detail page section gaps `gap-6` |
| xl | 32px | Layout gaps, page-level vertical rhythm |
| 2xl | 48px | Major section breaks, journey timeline card spacing |
| 3xl | 64px | Page-level spacing, sidebar expanded width (256px = 4*64), header height (64px = h-16) |

Exceptions:
- Touch targets: minimum 44px for icon-only buttons on mobile (accessibility)
- Pipeline columns: 256px width (`w-64`) for kanban columns
- Bottom sheet: `max-h-[90vh]` for mobile sheet overlays

Source: CONTEXT.md (sidebar 256px / 64px collapsed, header 64px), platform layout `gap-4 p-4`

## Color

All colors use oklch via CSS custom properties. Never use raw color values.

| Role | CSS Variable | Light Value | Dark Value | Usage |
|------|-------------|-------------|------------|-------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` | `oklch(0.145 0 0)` | Page background, main surfaces |
| Secondary (30%) | `--card` / `--muted` / `--sidebar` | `oklch(1 0 0)` / `oklch(0.97 0 0)` / `oklch(0.985 0 0)` | `oklch(0.205 0 0)` / `oklch(0.269 0 0)` / `oklch(0.205 0 0)` | Cards, sidebar, nav, skeleton fills, empty state backgrounds |
| Accent (10%) | `--primary` | `oklch(0.505 0.213 27.518)` | `oklch(0.444 0.177 26.899)` | See reserved-for list below |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` | `oklch(0.704 0.191 22.216)` | Destructive actions only |

Accent reserved for:
- Primary CTA buttons (`bg-primary`)
- Sidebar active item highlight (`--sidebar-accent`)
- Journey timeline active module card border
- Badge variant for active/current status
- Focus ring on interactive elements (`--ring`)
- Progress bar fill
- Chart accent lines (`--chart-1` through `--chart-5`)

Destructive reserved for:
- Delete/cancel confirmation buttons (`variant="destructive"`)
- Error toasts (sonner error style)
- Error page crack illustration stroke (`stroke-destructive/60`)
- Form validation error text

Semantic classes to use (never raw):
- `bg-background`, `text-foreground` -- primary surface
- `bg-card`, `text-card-foreground` -- card surfaces
- `bg-muted`, `text-muted-foreground` -- subdued text, skeleton fills, secondary content
- `bg-primary`, `text-primary-foreground` -- accent buttons
- `bg-destructive`, `text-destructive` -- destructive actions

Dark mode audit requirement: WCAG AA contrast (4.5:1 minimum for body text). All screens must be navigated in both themes during implementation.

Source: `packages/ui/src/styles/globals.css`, CONTEXT.md (dark mode audit, WCAG AA)

## Interaction Contracts

### Loading States

| Screen Type | Skeleton Component | Props | Duration |
|-------------|-------------------|-------|----------|
| DataTable lists | `TableSkeleton` | `rows={8}` | 800ms dev delay via `useSimulatedLoading()` |
| Dashboard card grids | `CardGridSkeleton` | `cards={4}` | 800ms dev delay |
| Pipeline boards | `PipelineSkeleton` | `stages={5}` for CRM, `stages={4}` for others | 800ms dev delay |
| Detail pages | `DetailSkeleton` | default | 800ms dev delay |

Animation: Pulse (shadcn default). Skeleton dimensions match real component dimensions exactly.

Form submission: `isSubmitting` from react-hook-form disables button and shows spinner.

### Responsive Breakpoints

| Breakpoint | Width | Layout Changes |
|------------|-------|----------------|
| Mobile | < 768px (`md`) | Single column, stacked layouts, horizontal tab pipelines, bottom sheets, reduced font sizes, horizontal scroll toolbar |
| Desktop | >= 768px | Multi-column, side-by-side layouts, kanban boards, dialog modals, full font sizes |

Detection: `useIsMobile()` hook from `@workspace/ui/hooks/use-mobile` (matchMedia at 768px).

### Responsive Pattern: Tables

- Horizontal scroll enabled on mobile
- Less-important columns hidden below `md` breakpoint via `hidden md:table-cell`
- Badge overflow: max 3 visible + "+N" tooltip via `BadgeOverflow` component
- Column visibility is fixed (no user column selector)

### Responsive Pattern: Pipelines

- Desktop: Kanban board with `@hello-pangea/dnd` drag-and-drop
- Mobile (< 768px): Horizontal `Tabs` per stage with card list per tab, no drag-and-drop
- Stage change on mobile: via select dropdown in detail page
- Tab triggers show stage label + counter: "{Stage} ({count})"

### Responsive Pattern: Detail Pages

- Desktop: 2-column grid (`lg:grid-cols-3`, main content `lg:col-span-2` + sidebar)
- Mobile (< 768px): Single column stack, actions/timeline first, main data below

### Responsive Pattern: Modals and Overlays

- Desktop: `Dialog` component with sizes `md` (448px), `lg` (512px), `xl` (preview)
- Mobile (< 768px): `Sheet` with `side="bottom"`, `max-h-[90vh]`, `overflow-y-auto`
- Always include `SheetTitle` (use `className="sr-only"` if visually hidden)

### Navigation and Transitions

| Behavior | Implementation | Duration |
|----------|---------------|----------|
| Page transition | `NavigationTransition` wrapper (fade/slide) | 150ms |
| Scroll reset | Automatic on route change | Instant |
| Dialog/modal open | shadcn default animation | 300ms |
| State change | CSS transition | 200ms |
| Micro-interaction (hover, focus) | CSS transition | 150ms |
| Tooltip show delay | `TooltipProvider delayDuration={300}` | 300ms |
| Tooltip hide | Instant on mouse leave | 0ms |

### Journey Page (`/jornada`)

- Horizontal timeline with cards connected by arrow pseudo-elements
- Card structure: module icon + module name + active item counter
- Order: Orcamento -> CRM -> Anteprojeto -> Proposta -> Projeto -> Obra
- Empty modules (0 active items): hidden from timeline
- Built with pure CSS/Tailwind + Flexbox (no diagram library)
- Sidebar entry: "Jornada" in OPERACAO group
- Mobile: horizontal scroll or vertical stack

### Lineage and Navigation

- "Registro" section in detail page sidebar: links to related entities (origin, next)
- "Voltar" button in detail page header: returns to previous entity in chain
- "Proxima etapa" buttons: "Criar oportunidade" (from Orcamento), "Gerar proposta" (from Anteprojeto), etc.
- Dashboard: consolidated pendencies panel with links to entities across the journey

### Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Focus trap | Dialogs and Sheets trap focus (first field on open, return to trigger on close) |
| Labels | All inputs have associated `<label>` elements |
| Aria labels | Icon-only buttons have `aria-label` |
| Visible focus | Focus ring visible on all interactive elements (via `--ring` token) |
| Contrast | WCAG AA (4.5:1) for body text in both themes |

### Data Formatting

| Format | Pattern | Implementation |
|--------|---------|---------------|
| Long text | Truncate + ellipsis + tooltip on hover | `truncate` class + `Tooltip` |
| Dates | DD/MM/YYYY | `Intl.DateTimeFormat("pt-BR")` |
| Date + time | DD/MM/YYYY HH:mm | `Intl.DateTimeFormat("pt-BR", { dateStyle: "short", timeStyle: "short" })` |
| Currency | R$ X.XXX,XX | `Intl.NumberFormat("pt-BR", { style: "currency", currency: "BRL" })` |
| Avatar fallback | Colored initials | `AvatarFallback` with color derived from name |

### Form Conventions

| Pattern | Rule |
|---------|------|
| Toggle (binary) | `Switch` component |
| Multi-select | `Checkbox` component |
| 2-4 options | `RadioGroup` component |
| 5+ options | `Select` component |
| Long text | `Textarea` with auto-resize up to 5 lines, then scroll |
| Required fields | Asterisk (*) next to label |
| Defaults | Empty fields (except inherited data from previous entity) |
| Disabled buttons | Visible with reduced opacity, never hidden |
