## UAT Gaps Being Closed

### Gap 1: Filter "Responsavel" UX Issue (Test 5)
- **User reported:** "nao tem filtro: selecione um Responsavel"
- **Severity:** major
- **Root cause:** The Select component shows "Todos os responsaveis" when value is "all", but the user expected a prompt to select a responsible person. The placeholder "Responsavel" only shows when no value is set.
- **Fix:** Change the "all" option label from "Todos os responsaveis" to "Selecione um Responsavel" to make the filter action more clear, or add a dedicated placeholder that shows when "all" is selected.

### Gap 2: Price Item Detail Dialog Height (Test 11)
- **User reported:** "modal de detalhes do item esta abrindo muito grande verticalmente, precisa ajustar"
- **Severity:** cosmetic
- **Root cause:** DialogContent has max-width but no max-height constraint. When pricing table has many rows (5 regions x 5 bands = up to 25 rows), the dialog grows too tall.
- **Fix:** Add max-height with overflow-y-auto to the dialog content area.
</context>
